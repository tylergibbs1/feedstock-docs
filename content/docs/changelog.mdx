## v0.1.6

**Auto-escalation on anti-bot blocks**

- **Auto-escalate on 403/429/503** ‚Äî when FetchEngine gets a blocked response (Cloudflare challenge, "Access Denied", rate limit), automatically escalates to PlaywrightEngine with stealth. No manual `withRetry()` needed.
- **`autoEscalateOnBlock`** config option on `EngineManager` (default: true). Disable with `{ autoEscalateOnBlock: false }`.
- **SPA detection scoped to 2xx responses** ‚Äî prevents false escalation on error pages with short bodies.
- 7 new unit tests for escalation logic (403, 429, 503, disable flag, fallback without browser, SPA + block sequence).

## v0.1.5

**Built-in stealth mode**

- **`stealth: true`** on `BrowserConfig` ‚Äî one flag enables random user-agents (9 realistic browser UAs), `navigator.webdriver` override, Chrome runtime spoofing, plugin/language spoofing via `addInitScript`
- **`simulateUser: true`** on `CrawlerRunConfig` ‚Äî automatically runs random mouse movements and scrolling after navigation. No manual hooks needed.
- Stealth is applied at context creation, so every page in the session gets it automatically

## v0.1.4

**Production hardening**

- **Input validation** ‚Äî `crawl()`, `crawlMany()`, `deepCrawl()` validate URLs upfront with friendly error messages
- **FetchEngine retry** ‚Äî retries transient network errors (ECONNRESET, ETIMEDOUT) up to 2 times with backoff
- **Session limits** ‚Äî `BrowserManager` caps at 20 concurrent sessions with LRU eviction
- **Cache pruning** ‚Äî `cache.pruneOlderThan(ms)` and `cache.size` for TTL-based cleanup
- **Graceful shutdown** ‚Äî SIGINT/SIGTERM handlers auto-close browser on process exit
- **User-agent rotation** ‚Äî `UserAgentRotator` with 9 realistic browser user-agents, `getRandomUserAgent()`
- **Static interactive detection** ‚Äî `detectInteractiveElementsStatic()` works without browser for `processHtml`
- **`processHtml` wired** ‚Äî `detectInteractiveElements` config option now works on raw HTML
- **prepublishOnly** ‚Äî typecheck + lint + unit tests run before every `npm publish`

## v0.1.3

**Performance optimizations and Playwright-native features**

### Performance
- **HTMLRewriter streaming extraction** ‚Äî links, media, and metadata extracted via Bun's native streaming parser instead of Cheerio DOM. Zero DOM allocation for the extraction path.
- **`scrapeAll()` single-pass** ‚Äî reduced Cheerio parses from 4x to 1x per page
- **BM25 frequency maps** ‚Äî O(1) term lookup via pre-computed frequency maps and Sets
- **Static cheerio import** ‚Äî removed dynamic `require()` in snapshot module
- **Batch `pathToRegex`** ‚Äî single regex replace instead of char-by-char loop (-10 lines)
- **Simplified engine selection** ‚Äî removed dead code in `selectEngines` (-6 lines)
- **LCS hash-based default** ‚Äî lowered DP threshold to 10K for faster diffs on large pages

### New Features
- **`blockResources`** ‚Äî abort images, CSS, fonts, and media during browser crawls via `context.route()`. Can cut page load 50-80%.
- **`navigationWaitUntil`** ‚Äî configurable navigation strategy: `"commit"` (fastest), `"domcontentloaded"` (default), `"load"`, `"networkidle"`
- **`extractInPage()`** ‚Äî extract links/media/metadata directly inside the browser via `page.evaluate()`, eliminating HTML serialization round-trip
- **`extractAllStreaming()`** ‚Äî Bun HTMLRewriter-based extraction as a standalone function
- **Service worker blocking** ‚Äî `serviceWorkers: 'block'` on every browser context ensures route interception works on all sites

## v0.1.1

**Agent-browser features and fetch-first engine system**

### Engine System
- **Fetch-first architecture** ‚Äî `FetchEngine` (HTTP) tried before `PlaywrightEngine` (browser)
- Auto-escalation: detects SPA shells (React, Next.js, Nuxt) and switches to browser
- `EngineManager` with quality-scored engine selection and fallback chain
- `likelyNeedsJavaScript()` heuristic for SPA detection

### Accessibility Snapshots
- `buildStaticSnapshot()` ‚Äî Cheerio-based semantic tree extraction (works with any engine)
- `takeSnapshot()` ‚Äî CDP-based `Accessibility.getFullAXTree` for browser-precise trees
- Node categorization: interactive (button, link, input), content (heading, paragraph, img), structural (filtered)
- `@e` ref system for deterministic element identification
- New `snapshot` field on `CrawlResult`, enabled via `config.snapshot = true`

### Rich Metadata (50+ fields)
- Full Open Graph (12 fields), Twitter Card (7), Dublin Core (7), Article (5)
- JSON-LD parsing, favicons, RSS/Atom feeds, alternate hreflang links
- Charset, viewport, theme-color, robots, referrer, generator
- Null values auto-stripped for cleaner output

### Filter Denial Reasons
- `applyWithReason()` on every filter returns `{ allowed, reason, filter }`
- Specific reasons: "Domain X is blocked", "Matched exclude pattern: Y", "File extension .pdf is blocked"
- `FilterChain.getDenials()` and `getDenialsByFilter()` for aggregate tracking
- Fully backward compatible ‚Äî `apply()` still returns boolean

### Browser Utilities
- **Interactive element detection** ‚Äî `detectInteractiveElements()` finds all clickable elements via single JS evaluation (cursor:pointer, onclick, tabindex, ARIA roles)
- **Iframe content inlining** ‚Äî `extractIframeContent()` + `inlineIframeContent()`
- **Storage state persistence** ‚Äî `saveStorageState()` / `loadStorageState()` for cookies + localStorage

### AI-Friendly Errors
- `toFriendlyError()` converts 20+ error patterns (DNS, timeout, SSL, element interaction, browser crashes) into actionable messages
- `withFriendlyErrors()` wrapper for any async operation
- Auto-applied in `crawler.crawl()` error handler

### Testing
- 277 unit/integration tests (up from 191)
- 115 dogfood checks against real websites
- Battle tests: engine fallback, redirects, timeouts, 404s, cache modes, screenshots, custom JS, network capture

## v0.1.0

**Initial release ‚Äî full-featured web crawler for TypeScript/Bun**

### Core Crawling
- `WebCrawler` with `crawl()`, `crawlMany()`, `processHtml()` methods
- `BrowserConfig` and `CrawlerRunConfig` with typed defaults
- Concurrent crawling with configurable concurrency limit
- Context manager pattern with `start()`/`close()` lifecycle

### Engine System
- **Fetch-first architecture**: tries lightweight HTTP fetch before launching a browser
- `FetchEngine` (quality 5) ‚Äî simple HTTP, no browser overhead
- `PlaywrightEngine` (quality 50) ‚Äî full Chromium/Firefox/WebKit
- Auto-escalation: detects SPA shells (React, Next.js, Nuxt) and switches to browser
- `EngineManager` with quality-scored engine selection

### Browser Backends
- **Playwright** ‚Äî Chromium, Firefox, WebKit
- **Lightpanda** ‚Äî local mode via `@lightpanda/browser`, cloud mode via CDP WebSocket

### Content Processing
- HTML cleaning via Cheerio (strips scripts, styles, noise tags)
- Link extraction with internal/external classification
- Media extraction (images, videos, audio) with quality scoring
- **Rich metadata extraction** ‚Äî 50+ fields: Open Graph, Twitter Cards, Dublin Core, JSON-LD, article tags, favicons, feeds, alternates
- Markdown generation via Turndown with citation support
- Accessibility tree snapshots ‚Äî compact semantic page representation with `@e` refs

### Extraction Strategies
- **CSS selector extraction** ‚Äî map selectors to JSON fields
- **Regex extraction** ‚Äî pattern matching with named capture groups
- **XPath extraction** ‚Äî XPath-to-CSS conversion
- **Table extraction** ‚Äî structured headers, rows, captions

### Deep Crawling
- **BFS** (Breadth-First Search) ‚Äî level-by-level with concurrent batching
- **DFS** (Depth-First Search) ‚Äî single-path depth exploration
- **BestFirst** ‚Äî score-based priority queue using composite scorers
- Streaming mode via `deepCrawlStream()` async generator
- `maxDepth`, `maxPages`, `concurrency` controls

### URL Filtering
- `URLPatternFilter` ‚Äî glob/regex include/exclude patterns
- `DomainFilter` ‚Äî whitelist/blacklist domains
- `ContentTypeFilter` ‚Äî extension-based filtering
- `MaxDepthFilter` ‚Äî depth limit per URL
- `FilterChain` ‚Äî composable, short-circuit evaluation
- **Denial reasons** ‚Äî track why each URL was rejected with `getDenials()` / `getDenialsByFilter()`

### URL Scoring
- `KeywordRelevanceScorer` ‚Äî match keywords in URL and anchor text
- `PathDepthScorer` ‚Äî shallower paths score higher
- `FreshnessScorer` ‚Äî URLs with recent dates score higher
- `DomainAuthorityScorer` ‚Äî preferred domains score highest
- `CompositeScorer` ‚Äî weighted averaging of multiple scorers

### Caching
- SQLite-based cache via `bun:sqlite` with WAL mode
- 5 cache modes: Enabled, Disabled, ReadOnly, WriteOnly, Bypass
- `CacheValidator` ‚Äî HTTP HEAD requests with ETag/Last-Modified
- Batch insert via `setMany()` (atomic transactions)

### Rate Limiting & Compliance
- Per-domain rate limiter with exponential backoff on 429/503
- Gradual recovery on success
- Configurable jitter, max delay, backoff/recovery factors
- **Robots.txt parser** ‚Äî User-agent matching, Allow/Disallow, Crawl-delay, Sitemap discovery, wildcard patterns

### Anti-Bot
- `isBlocked()` ‚Äî detects Cloudflare challenges, CAPTCHAs, 403/429/503 blocks
- `applyStealthMode()` ‚Äî overrides navigator.webdriver, plugins, languages
- `simulateUser()` ‚Äî random mouse movements and scrolling
- `withRetry()` ‚Äî automatic retry with escalating delays

### Content Filtering
- `PruningContentFilter` ‚Äî rule-based boilerplate removal
- `BM25ContentFilter` ‚Äî relevance-based filtering by query

### Chunking
- `RegexChunking` ‚Äî split by patterns (default: paragraphs)
- `SlidingWindowChunking` ‚Äî word-count windows with overlap
- `FixedSizeChunking` ‚Äî character-count chunks with overlap
- `IdentityChunking` ‚Äî no splitting

### Browser Utilities
- **Interactive element detection** ‚Äî finds all clickable elements including cursor:pointer, onclick, tabindex
- **Iframe content inlining** ‚Äî extracts and merges iframe content into parent HTML
- **Storage state persistence** ‚Äî save/load cookies and localStorage between sessions
- **Hooks** ‚Äî onPageCreated, beforeGoto, afterGoto, onExecutionStarted, beforeReturnHtml

### Infrastructure
- **Proxy rotation** ‚Äî round-robin with health tracking and auto-recovery
- **URL seeder** ‚Äî sitemap discovery via robots.txt chain
- **Crawler monitor** ‚Äî real-time stats (pages/sec, success rates, data volume)
- **AI-friendly errors** ‚Äî converts 20+ error patterns into actionable messages
- **Logging** ‚Äî ConsoleLogger with level filtering, SilentLogger, pluggable Logger interface

### Developer Experience
- Native TypeScript execution via Bun (no build step)
- 260 tests via `bun:test`
- Biome for linting and formatting
- GitHub Actions CI
- Apache-2.0 license
